# AI Code Marker 操作文档

## 1. 作用

这个工具做两件事：

- 记录本次 staged 新增行里，哪些行属于 AI 生成
- 在 `git commit` 时自动把 AI 行数、总行数、占比和 AI 类型写入 commit message

它不依赖提示词，并会在提交前自动把 AI 代码块写成 `AI-GENERATED-BEGIN/END` 注释标记。
归因数据会在提交完成后写入 `git notes` 的 `refs/notes/ai`。

## 2. 记录方式

参考 `git-ai` 的思路，归因分两步：

1. 提交前：把当前 staged 的 AI 新增行范围记录到 `.git/ai-code-marker/staged-attribution.json`
2. 提交后：`post-commit` hook 自动把这份归因固化到 `refs/notes/ai`

当前 note 结构示例：

```json
{
  "version": 1,
  "tool": "AI",
  "model": null,
  "files": [
    {
      "path": "src/app.py",
      "added_lines": [10, 11, 12]
    }
  ]
}
```

## 3. 安装

前提：

- 已安装 `git`
- 已安装 `python 3.10+`

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\install-hooks.ps1
```

给别的 Git 仓库安装：

```powershell
powershell -ExecutionPolicy Bypass -File .\install-hooks.ps1 -RepoRoot C:\path\to\target-repo
```

macOS/Linux:

```bash
sh ./install-hooks.sh
```

给别的 Git 仓库安装：

```bash
sh ./install-hooks.sh /path/to/target-repo
```

通用方式：

```bash
python setup.py develop
python -m ai_code_marker.cli install-hook --repo-root .
```

安装到别的 Git 仓库：

```bash
python setup.py develop
python -m ai_code_marker.cli install-hook --repo-root /path/to/target-repo
```

安装后会生成三个 hook：

- `pre-commit`
- `prepare-commit-msg`
- `post-commit`

hook 会固定使用安装时的 Python 解释器执行 `python -m ai_code_marker.cli`，因此不再依赖目标仓库里的 `src/ai_code_marker/cli.py`，也不需要在目标仓库提交时设置 `PYTHONPATH`。
其中 `pre-commit` 会在提交前自动把本次 staged 的 AI 代码块包上 `AI-GENERATED-BEGIN/END` 注释，并自动重新 `git add`。Codex 和 Claude Code 使用同一套 hook 记录行数，不需要为不同开发工具手工指定工具名。

## 4. 使用流程

日常流程：

1. 完成代码修改
2. 执行 `git add`
3. 执行 `git commit`
4. `pre-commit` 自动补写 `AI-GENERATED-BEGIN/END`
5. `prepare-commit-msg` 自动写入 AI 统计
6. `post-commit` 自动把归因信息写入 `refs/notes/ai`

默认工具名为通用的 `AI`。如果确实需要区分工具，可以在提交前设置：

Windows PowerShell:

```powershell
$env:AI_CODE_MARKER_TOOL = "Claude Code"
$env:AI_CODE_MARKER_MODEL = "gpt-5.4"
```

macOS/Linux:

```bash
export AI_CODE_MARKER_TOOL="Claude Code"
export AI_CODE_MARKER_MODEL=gpt-5.4
```

提交时 commit message 会被自动改写为：

```text
feat: add api

AI生成代码行数：[12]，总提交代码行数：[40]，占比：30.00%，AI协同
```

提交完成后，可直接查看 note：

```bash
git notes --ref refs/notes/ai show HEAD
```

## 5. 手工查看统计

Windows PowerShell:

```powershell
python -m ai_code_marker.cli stats --staged --json
```

macOS/Linux:

```bash
python -m ai_code_marker.cli stats --staged --json
```

## 6. 按时间范围统计

默认统计范围是“当前时间回溯到本周一零点”，默认仓库是当前目录。

Windows PowerShell:

```powershell
python -m ai_code_marker.cli range-report --json
```

macOS/Linux:

```bash
python -m ai_code_marker.cli range-report --json
```

指定时间范围和仓库：

```bash
python -m ai_code_marker.cli range-report --repo-root /path/to/repo --since 2026-04-21T00:00:00+08:00 --until 2026-04-22T23:59:59+08:00 --json
```

## 7. 当前口径

- `AI-Code-Lines`：当前 staged 中，被 AI 归因覆盖到的新增非空行
- `AI-Total-Lines`：当前 staged 中全部新增非空行
- `AI-Code-Ratio`：`AI-Code-Lines / AI-Total-Lines`

注意：

- 目前只统计 staged 的新增非空行，不统计删除行
- 目前默认会在 `pre-commit` 把当前 staged 的新增代码块记为 AI，并自动写入源码标记
- 提交完成后，pending 归因文件会被清理，只保留 commit note

## 8. 推荐接入方式

- 团队统一安装 hook，保证 commit message 和 note 一致生成
- 后续如果要继续向 `git-ai` 靠拢，可以在此基础上增加按文件区间查询、prompt/session 追踪和 continue/search 能力
