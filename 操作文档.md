# AI Code Marker 操作文档

## 1. 作用

这个工具做两件事：

- 记录本次 staged 新增行里，哪些行属于 AI 生成
- 在 `git commit` 时自动把 AI 行数、总行数、占比、工具名、文件名写入 commit message

它不依赖提示词，也不要求在源码里插入注释标记。
归因数据会在提交完成后写入 `git notes` 的 `refs/notes/ai`。

## 2. 记录方式

参考 `git-ai` 的思路，归因分两步：

1. 提交前：把当前 staged 的 AI 新增行范围记录到 `.git/ai-code-marker/staged-attribution.json`
2. 提交后：`post-commit` hook 自动把这份归因固化到 `refs/notes/ai`

当前 note 结构示例：

```json
{
  "version": 1,
  "tool": "Codex",
  "model": "gpt-5.4",
  "files": [
    {
      "path": "src/app.py",
      "added_lines": [10, 11, 12]
    }
  ]
}
```

## 3. 安装

前提：

- 已安装 `git`
- 已安装 `python 3.10+`

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\install-hooks.ps1
```

macOS/Linux:

```bash
sh ./install-hooks.sh
```

通用方式：

```bash
PYTHONPATH=./src python -m ai_code_marker.cli install-hook --repo-root .
```

安装后会生成两个 hook：

- `prepare-commit-msg`
- `post-commit`

## 4. 使用流程

日常流程：

1. 完成代码修改
2. 执行 `git add`
3. 对当前 staged 改动记录 AI 归因
4. 执行 `git commit`

记录 staged AI 归因：

Windows PowerShell:

```powershell
$env:PYTHONPATH = ".\\src"
python -m ai_code_marker.cli record-staged --tool Codex --model gpt-5.4
```

macOS/Linux:

```bash
PYTHONPATH=./src python -m ai_code_marker.cli record-staged --tool Codex --model gpt-5.4
```

提交时 commit message 会被自动改写为：

```text
[AI协同](ai:12/total:40/ratio:30.00%) feat: add api

AI-Code-Lines: 12
AI-Total-Lines: 40
AI-Code-Ratio: 30.00%
AI-Tools: Codex
AI-Files: src/api.py
```

提交完成后，可直接查看 note：

```bash
git notes --ref refs/notes/ai show HEAD
```

## 5. 手工查看统计

Windows PowerShell:

```powershell
$env:PYTHONPATH = ".\\src"
python -m ai_code_marker.cli stats --staged --json
```

macOS/Linux:

```bash
PYTHONPATH=./src python -m ai_code_marker.cli stats --staged --json
```

## 6. 当前口径

- `AI-Code-Lines`：当前 staged 中，被 AI 归因覆盖到的新增非空行
- `AI-Total-Lines`：当前 staged 中全部新增非空行
- `AI-Code-Ratio`：`AI-Code-Lines / AI-Total-Lines`

注意：

- 目前只统计 staged 的新增非空行，不统计删除行
- 目前的 `record-staged` 默认把当前 staged 全部新增非空行记为 AI 行
- 提交完成后，pending 归因文件会被清理，只保留 commit note

## 7. 推荐接入方式

- 把 `record-staged` 接到你的 AI 工具工作流里，在 AI 完成一次改动并 `git add` 后自动执行
- 团队统一安装 hook，保证 commit message 和 note 一致生成
- 后续如果要继续向 `git-ai` 靠拢，可以在此基础上增加按文件区间查询、prompt/session 追踪和 continue/search 能力
